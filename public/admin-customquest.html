カスタムクエスト管理 — 使い方ガイド

Step 1: テンプレートを作成する

テンプレートは「食事メニュー」または「運動メニュー」の雛形です。1回分の献立や1セッション分のトレーニングを登録します。

  1. 左サイドバー上部の + 新規 をクリック
  2. 右側に編集パネルが開くので テンプレート名 を入力（例: 【減量】朝食セット）
  3. タイプ を MEAL（食事） or WORKOUT（運動）から選択
  4. カテゴリタブ で食品/種目を絞り込み、検索欄で素早く見つける
  5. アイテムを + で追加。追加後も量・単位・セット数等を直接編集可
  6. テンプレートを保存 をクリック
左サイドバー下部の「食事テンプレ一括」「運動テンプレ一括」で標準テンプレを自動生成することもできます。

Step 2: ユーザーを選択する

  1. 中央エリア上部の入力欄に ユーザーID（アプリの設定画面で確認できます） を入力
  2. 読み込み をクリック
  3. ユーザーのプロフィール・ルーティン・PFCサマリー・スロットタイムラインが表示される
一度読み込んだユーザーは「最近のユーザー」にボタンとして残るので、ワンクリックで切替できます。

Step 3: スロットにテンプレートを割り当てる

スロット＝1日の食事・運動枠です。例: 5食+1トレーニング ＝ 6スロット。

  1. スロットタイムライン の各カードにあるドロップダウンからテンプレートを選択
  2. テンプレート選択時、カロリーがスロット目標に自動スケーリングされる（食事のみ）
  3. PFCサマリーがリアルタイムで更新 — 過不足をバーで確認
  4. 全スロット設定後、一括保存 で保存
永続 / 日付指定の違い:
  • 永続 — 毎日適用されるデフォルトプラン（_default として保存）
  • 日付指定 — 特定日のみ上書き（その日だけの特別プラン）

Step 4: PFCバーの見方

PFCサマリーの各バーは、ユーザーの目標に対する割当量の割合を表示します。

ブルー (95-105%): 目標達成
オレンジ (<95%): やや不足
赤 (<70%): 大幅不足
紫 (>105%): 超過

コーチング・セオリー

基本哲学
  • 食べなければ身体は作れない — トレーニングはきっかけ、身体を作るのは食事
  • 365日の継続 — オン・オフを作らず淡々と続ける
  • 外食より自炊 — 見えない脂質・調味料を排除
PFC戦略 — P35% F15% C50%
P = エンジン
筋肉の材料 / 最重要
C = ガソリン
筋肉を動かす / 抜くとハリ喪失
F = 最小限
9kcal/gで過剰注意 / 必須脂肪酸はサプリで
GIタイミング戦略
タイミングGI推奨食材理由
トレ直後高GIマルトデキストリン、白米（炊飯直後）、もち、バナナインスリン→筋肉へ急速デリバリー
トレ前低GI白米（冷やご飯・再加熱）、オートミール、さつまいも + 低脂質消化遅延防止＆持続エネルギー
通常食低GI白米（冷やご飯・再加熱）、玄米、オートミール、そば血糖安定＆体脂肪化防止
冷却でレジスタントスターチ生成（RS: 0.64→1.65g/100g）。再加熱してもRSは維持されGI低下（88→65）。炊飯直後はトレ直後専用
ベースライン法（炭水化物量の調整）
  1. 基準量（ベースライン）を決めて1週間食べる
  2. 体重増加 → 炭水化物を半分に減らす
  3. 体重減少しすぎ → 炭水化物を倍に増やす
  4. 微調整を繰り返し、体脂肪が増えないギリギリのラインを探す
「減らしすぎない」— 代謝を落とさず筋肉のハリを保つ

テンプレートの編集・複製・削除

左サイドバーのテンプレートカードにカーソルを当てると、操作ボタンが表示されます。

  • 編集 — 右側に編集パネルが開き、名前・アイテムの変更が可能
  • 複製 — 同じ内容で新しいテンプレートを作成
  • 削除 — テンプレートを非表示に（確認ダイアログあり）

プロフィール編集

中央エリア「プロフィール」カード右上のタブで表示と編集を切り替えます。

  • 表示 — 現在のプロフィール情報を確認（読み取り専用）
  • 時刻 — 起床/就寝/食事回数/トレーニング時間帯
  • 基本 — 身長/体重/体脂肪率/理想体型
  • 目標 — 減量/維持/増量 + PFC比率 + トレ前後PFC固定値
  • 食材 — 優先食材/NG食材/アレルギー
  • トレ加算 — 自動算出(編集不要)。分割タイプから3段階Class(SSS/S/A)を判定し、LBMでスケーリング

編集後は 保存 を忘れずに。

Tips

  • ESCキーで編集パネルを閉じることができます
  • ヘッダーの「← 管理画面」で通常の管理画面に戻れます
  • テンプレートの量変更時は追加済みアイテムの数値を直接クリックして編集できます
  • スロット割当を変更するたびにPFCバーがリアルタイムで更新されます

用語集

栄養・エネルギー
PFC
Protein(タンパク質) / Fat(脂質) / Carbs(炭水化物) の頭文字。3大栄養素の比率。アプリのデフォルトは P35% F15% C50%。P=4kcal/g、F=9kcal/g、C=4kcal/g で計算
カロリー
エネルギーの単位(kcal)。摂取カロリー < 消費カロリーで減量、逆で増量。本アプリでは全てkcal単位
GI値
Glycemic Index（血糖指数）。食後の血糖値上昇速度を示す指標。高GI=急上昇(白米炊きたて等)、低GI=緩やか(玄米、冷やご飯等)。トレ直後は高GI、それ以外は低GI推奨
DIAAS
Digestible Indispensable Amino Acid Score（消化性必須アミノ酸スコア）。タンパク質の「質」を評価する最新指標。アプリのミクロ+インジケーターで表示
RS
Resistant Starch（レジスタントスターチ）。冷却で生成される難消化性でんぷん。白米を冷やすとRS増加(0.64→1.65g/100g)、再加熱しても維持されGI低下(88→65)
身体・代謝
BMR
Basal Metabolic Rate（基礎代謝量）。何もしなくても消費するカロリー。本アプリではKatch-McArdle式を採用: BMR = 370 + 21.6 × LBM + 4.5 × 脂肪量
TDEE
Total Daily Energy Expenditure（1日の総消費カロリー）。TDEE = BMR × 活動係数。活動係数: デスクワーク1.2 / 立ち仕事1.4 / 肉体労働1.6
LBM
Lean Body Mass（除脂肪体重）。体重から体脂肪を除いた重さ。LBM = 体重 × (1 - 体脂肪率/100)。トレ加算やBMRの計算基準
MET
Metabolic Equivalent of Task（代謝当量）。運動の強度を安静時の倍数で表す。脚トレ=6.0MET、胸/背中=5.0MET、腕=4.0MET。アプリでは記録表示用であり、カロリー収支計算には使用しない
EPOC
Excess Post-exercise Oxygen Consumption（運動後過剰酸素消費）。トレーニング後も代謝が上がり続ける現象。大筋群ほどEPOCが大きい → Class SSSの根拠
トレ加算・ルーティン
トレ加算
トレーニング日のリカバリー(回復)に必要なエネルギー予算。TDEEに上乗せされる。分割タイプから3段階Classで自動算出、LBMでスケーリング
Class SSS
+400kcal(LBM60kg基準)。脚・全身・下半身。大筋群・高EPOC・高神経系疲労
Class S
+250kcal(LBM60kg基準)。胸・背中・肩・上半身・プッシュ・プル・複合系。標準的な筋肥大トレーニング
Class A
+100kcal(LBM60kg基準)。腕・腹筋/体幹。小筋群・回復コスト低
分割
1週間のトレーニングを部位ごとに分けるパターン。例: Day1=胸、Day2=背中、Day3=脚、Day4=肩、Day5=腕、Day6-7=休み
ルーティン
7日周期の分割パターン。アプリのルーティン設定で登録し、ダッシュボードで「今日の部位」を自動表示
アプリ機能・管理用語
カスタムクエスト
管理者がユーザーごとに設定する食事・運動プラン。テンプレートをスロットに割り当てて1日のメニューを構成する
テンプレート
食事/運動メニューの雛形。1食分の献立や1セッション分のトレーニングを食品・種目の組み合わせで定義
スロット
1日の食事・運動の「枠」。例: 5食+1運動=6スロット。各スロットに時刻とテンプレートを割り当てる
永続
毎日適用されるデフォルトの割当。日付指定の割当がない日に自動適用される
スケーリング
テンプレートのPFCをスロットの目標カロリーに合わせて自動比例調整すること
通常クエスト
Cloud Function(generateQuest)がルールベースで自動生成する翌日の食事・運動プラン。プレミアム会員限定。プロフィール・ルーティン・PFC目標から最適なメニューを提案。ダッシュボードの「クエスト生成」ボタンで実行
ミクロ+
ダッシュボードに表示される栄養品質インジケーター。DIAAS(アミノ酸スコア)、脂肪酸バランス、食物繊維、ビタミン/ミネラルの4項目
コンディション
睡眠時間・睡眠品質・消化・集中力・ストレスの5項目からなる体調記録。履歴画面でグラフ確認可能。スコアはコンディション評価のみ現在使用中
計算フロー
1. LBM = 体重 × (1 - 体脂肪率/100)
2. BMR = 370 + 21.6×LBM + 4.5×脂肪量
3. TDEE = BMR × 活動係数(1.2/1.4/1.6)
4. 目標Cal = TDEE ± 目標調整(±300) + トレ加算(Class×LBM/60)
5. P = 目標Cal × P% / 4、F = 目標Cal × F% / 9、C = 目標Cal × C% / 4
